========================================================
EVIDENCE 02 — Validation Errors (400)
========================================================
Test Date: March 11, 2026
Pre-conditions: All three services running with seeded data.

========================================================
Student Service — port 8001
========================================================

--------------------------------------------------------
Test: POST student — missing name field → 400
--------------------------------------------------------

$ curl -s -X POST http://localhost:8001/api/students \
    -H "Content-Type: application/json" \
    -d '{"email":"noname@example.com"}'

HTTP/1.1 400 Bad Request
Content-Type: application/json

{"error":"VALIDATION_ERROR","message":"The name field is required."}

--------------------------------------------------------
Test: POST student — missing email field → 400
--------------------------------------------------------

$ curl -s -X POST http://localhost:8001/api/students \
    -H "Content-Type: application/json" \
    -d '{"name":"No Email"}'

HTTP/1.1 400 Bad Request
Content-Type: application/json

{"error":"VALIDATION_ERROR","message":"The email field is required."}

--------------------------------------------------------
Test: POST student — invalid email format → 400
--------------------------------------------------------

$ curl -s -X POST http://localhost:8001/api/students \
    -H "Content-Type: application/json" \
    -d '{"name":"Bad Email","email":"not-an-email"}'

Content-Type: application/json

{"error":"VALIDATION_ERROR","message":"The email field must be a valid email address."}
